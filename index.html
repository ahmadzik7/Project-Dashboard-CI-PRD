⚡ Problem-Solving Framework (DMAPM)
1. DEFINE
Apa masalahnya? Di mana? Kapan? Dampak ke produksi?
2. MEASURE
Frekuensi, downtime, estimasi kerugian. Data untuk prioritas.
3. ANALYZE
5-Why atau Fishbone. Root cause, bukan symptom.
4. PLAN
Buat project entry. Konfirmasi material & jadwal Engineering.
5. MONITOR
Update tiap 2-3 hari. Hambatan → log Issue → eskalasi.
📧 Template Komunikasi ke Engineering
REQUEST PERTAMA
Halo [Nama PIC Engineering], 🔧 ISSUE: [Deskripsi masalah] 📍 LOKASI: [Mesin/Area] 💥 DAMPAK: [Dampak ke produksi] Mohon konfirmasi: 1. Kebutuhan material (list + part no.) 2. Estimasi hari kerja 3. Jadwal availability tim Target: [tanggal] | EJO: [nomor] Terima kasih 🙏
FOLLOW-UP
Halo [Nama], Follow-up: [nama project / EJO] ✅ Issue sudah diidentifikasi ⏳ Menunggu info material & jadwal Ada update? cc: [Atasan jika perlu]
